Boot Properties

Each Logscape component uses the configuration from a boot.properties file which is located in the components home directory. The component will need to be bounced or restarted for any changes in this file to take effect.

VMWARGS
Property Value
-Xms Configure Min Heap Memory. Start Heap
-XmxConfigure Max Heap Memory
SYSPROPS
Property Value
agent.roleSee the page Zonese.g dev.Manager or dev.Indexer
vso.base.port=11000The base port from which port allocation will commence
web.app.port Default value 8080
vso.zone=LOGSCAPE1
logscape.session.timeout.days=7
net.iface-Dnet.iface allows a specific network interface to be specified, i.e. -Dnet.iface=eth1
net.name.based -Dnet.name.based=true make the system use the hostname instead of the ipAddress (support for NAT)
dashboard.heap -Ddashboard.heap controls heap/mem allocated to the WebDashboard. [512M] increased with concurrent use.Ideal[1256M] or more
logspace.heap -Dlogspace.heap control heap/mem allocated to the LogSpace Service [256M] Ideal[1256M] or more
aggspace.heap-Daggspace.heap controls heap/mem allocated to the Aggregation Engine. [512M] Ideal[1256M] or more
search.flush.ms -Dsearch.flush.ms (default 500ms) - interval when to pump aggregate events resulting from AggFunctions (count, avg etc)
replay.flush.ms-Dreplay.flush.ms (default 200ms) - determines latency when flushing raw-event data on a search
OTHER

PRIORITY

Can have the values LOW,NORMAL,HIGH

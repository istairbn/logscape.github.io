Alert Schedule

Alerts are scheduled with the familiar crontab syntax. Alerts can be executed retrospectively or in real-time.

0 */2 * * *

Trigger Search - Trigger Search is executed

Standard Alerts

The default behaviour of an alert is to run a search retrospectively specified by the crontab schedule. The time window of the saved search every the alert fires. This should reflect the crontab schedule to avoid any duplicated events being reported.

Real-time Alerts

Real-time alerts will keep a search open and any trigger events which occur will be caught in real-time and the appropriate action executed. The Alert Search is refreshed on the schedule specified. What this means is that any changes to the trigger search will be picked up at the next scheduled time.

Trigger Threshold

Under the Trigger Tab you can set the minimum number of events to trigger the Alert. If this left empty then the results of the trigger search are used with the trigger action.

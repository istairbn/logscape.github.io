Deploying Forwarders

Logscape has a flexible deployment architecture. Each component can be configured to collect data and provide search analytics for custom host and operating system configurations. The Manager, by default, runs the following services.

  • IndexStore
  • SyslogServer
  • Indexer
  • Raw Socket Server
    • The most common deployment scenario is using a Manager and a collection of Forwarders installed on different machines collecting data.

    A Forwarder will collect data from applications running on a host and 'ship' the data to the Manager. The Manager will then index the data and prepare it for visulization. This happens in realtime providing instantaneous feedback on changes in your environment

    Pros

    • Ideal for low spec machines
    Cons
    • does not scale
    • a powerful manager is required.
    • More forwarders requires a manager greater memory and cpu resources
